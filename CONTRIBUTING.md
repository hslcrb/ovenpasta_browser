# Ovenpasta 기여 가이드 (Contributing Guide)

Ovenpasta에 관심을 가져주셔서 감사합니다! 이 프로젝트는 오픈 소스로 개발되며, 여러분의 참여를 환영합니다.
기여를 시작하기 전에 아래의 가이드라인을 읽어주세요.

## 버그 리포트 / 기능 요청
* 새로운 이슈를 생성하기 전에 이미 등록된 유사한 이슈가 없는지 검색해 주세요.
* 버그를 제보할 때는 재현 방법과 발생 환경(OS, 컴파일러, GTK 버전 등)을 최대한 상세히 기술해 주세요.
* 기능 요청의 경우 프로젝트의 방향성("외부 라이브러리 없이 구현하기")에 부합하는지 고려해 주세요.

## 개발 가이드라인
* **코딩 스타일**: 기존 코드베이스의 C++ 스타일을 따라주세요. 명시적인 룰은 아직 없으나 탭 대신 스페이스 4칸 들여쓰기, Snake case 변수명 등을 지향합니다.
* **언어**: 커밋 메시지 작성 시 한글을 우선 사용해 주시고, 영어 병기를 권장합니다 (사용자 규칙: `모든 커밋 한영병기. 커밋 메시지에 한글 우선.`).
* **기능 단위 커밋**: 한 번의 커밋에 여러 작업이 섞이지 않도록(기능별로) 분리해서 커밋해 주세요 (사용자 규칙: `기능별로 커밋하기`).
* **마인드셋**: 단순히 코드를 짜는 것을 넘어서, 완성도 있는 프로덕트를 만드는 개발자의 마음가짐으로 구조와 성능에 대해 고민해 주세요 (사용자 규칙: `프로덕트 개발자로 임하기`).
* **표준 의존성 준수**: GTK를 제외한 다른 외부 라이브러리(libcurl, Gumbo 파서, OpenSSL 등)의 추가는 지양합니다.

## 기여 절차
1. 이 저장소를 포크(Fork)합니다.
2. 새로운 기능 브랜치를 생성합니다. (`git checkout -b feature/awesome-feature`)
3. 코드를 작성하고 규칙에 맞게 커밋합니다.
4. 원격 저장소에 브랜치를 푸시합니다. (`git push origin feature/awesome-feature`)
5. Pull Request(PR)를 작성하여 코드 리뷰를 요청합니다.

의문점이나 토론할 내용이 있다면 언제든 이슈로 남겨주세요!
